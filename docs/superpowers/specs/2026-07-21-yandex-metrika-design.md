# Подключение Яндекс.Метрики

## Цель

Подключить к публичному сайту автошколы счётчик Яндекс.Метрики `110924306` и собирать аналитику только после явного согласия посетителя на аналитические cookie.

## Выбранный подход

Использовать существующие `CookieConsentProvider` и `YandexMetrika`. Не вставлять сырой HTML-фрагмент во все страницы: корневой `layout.tsx` уже подключает компонент один раз для всего приложения.

После согласия `accepted` компонент:

- создаёт очередь `window.ym`;
- загружает `https://mc.yandex.ru/metrika/tag.js?id=110924306`;
- инициализирует счётчик `110924306`;
- включает `ssr`, `webvisor`, `clickmap`, `accurateTrackBounce` и `trackLinks`;
- передаёт текущие `referrer` и `url`;
- сохраняет настройку `ecommerce: "dataLayer"`, выданную Яндексом.

При выборе только необходимых cookie скрипт не загружается. Если согласие на аналитику отозвано, счётчик уничтожается, тег удаляется, а глобальная функция очищается.

Постоянный `<noscript>`-пиксель не добавляется, потому что при отключённом JavaScript пользователь не может дать согласие через существующий интерфейс, а пиксель отправил бы просмотр без согласия.

## Конфигурация

Идентификатор не является секретом, но хранится как публичная переменная окружения:

`NEXT_PUBLIC_YANDEX_METRIKA_ID=110924306`

Значение добавляется в `.env.example` и во все контексты Netlify. После изменения Netlify выполняет новую сборку, чтобы публичная переменная попала в клиентский код.

## Компоненты и поток данных

1. `CookieConsentProvider` читает сохранённый выбор посетителя.
2. `YandexMetrika` получает состояние согласия.
3. Без согласия внешних запросов к Метрике нет.
4. После согласия тег загружается и счётчик инициализируется один раз.
5. Через кнопку «Настройки cookie» пользователь может отозвать согласие.

## Ошибки и безопасность

Ошибка загрузки внешнего тега не должна ломать интерфейс или формы сайта. Счётчик не получает Telegram-токен, содержимое серверных переменных или данные заявок. Вебвизор запускается только после разрешения аналитики.

## Проверка

- Автотест подтверждает точный URL тега и параметры инициализации.
- Автотест подтверждает отсутствие тега без согласия.
- Автотест подтверждает удаление счётчика после отзыва согласия.
- Выполняются lint, typecheck, полный набор тестов и production-сборка.
- После публикации проверяется наличие тега на публичном сайте после принятия cookie.

## Вне текущей задачи

Настройка целей и событий конверсии в интерфейсе Яндекс.Метрики не входит в это изменение. Их можно добавить отдельно после появления первых данных.
